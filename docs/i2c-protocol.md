# I²C Protocol

The Weather Station project communicates with a Bosch BME280 environmental
sensor over an I²C bus. This document summarizes the protocol, explains how the
Bosch driver is integrated into the STM32 HAL, and highlights the practical
details that mattered during development.

---

# What is I²C?

I²C (Inter-Integrated Circuit) is a synchronous serial bus designed for
communication between integrated circuits on the same PCB.

Unlike SPI, every device shares only two signal lines.

| Signal | Purpose |
|---------|---------|
| SDA | Serial Data |
| SCL | Serial Clock |

The STM32 always acts as the **bus master**, while the BME280 operates as a
slave device.

---

# Bus Idle State

Both SDA and SCL are implemented as open-drain outputs.

Neither device actively drives the lines High.
Instead, external pull-up resistors keep both lines High whenever the bus is
idle.

```
Idle bus

SCL ────────────────────────────── High

SDA ────────────────────────────── High
```

Communication begins only when the master generates a Start Condition.

---

# Start Condition

Every transfer starts with a Start Condition.

A Start is generated by pulling SDA Low while SCL remains High.

```
SCL ────────────────┐──────────────
                    │
SDA ───────┐________│______________
           │
         Start
```

Every slave monitors the bus continuously and detects this transition.

---

# Stop Condition

A transmission finishes with a Stop Condition.

This time SDA changes from Low to High while SCL is High.

```
SCL ────────────────┐──────────────

SDA _________┌──────│──────────────
             │
            Stop
```

After a Stop Condition the bus becomes free again.

---

# Typical Register Read

Most sensors—including the BME280—store their measurements inside internal
registers.

Reading one therefore requires two phases.

### Phase 1

Tell the sensor which register should be read.

```
START

Slave Address + Write

Register Address
```

### Phase 2

Immediately restart communication without releasing the bus.

```
Repeated START

Slave Address + Read

Read Data

STOP
```

This second Start is called a **Repeated Start**.

---

# Why a Repeated Start?

Without a Repeated Start, another master could theoretically gain access to the
bus between the write and read operations.

The Repeated Start keeps ownership of the bus while changing only the transfer
direction.

The STM32 HAL performs this sequence automatically inside
`HAL_I2C_Mem_Read()`.

---

# ACK and NACK

Every transmitted byte is acknowledged by the receiver.

```
Byte 1

ACK

Byte 2

ACK

Byte 3

ACK

...

Last Byte

NACK

STOP
```

An ACK means

- byte received correctly

A NACK usually means

- receiver has no more data
- receiver rejects the transfer
- last byte has been received

---

# Read vs. Write

| Write | Read |
|--------|------|
| Master -> Slave | Slave -> Master |
| R/W bit = 0 | R/W bit = 1 |
| Usually one transaction | Usually write register address first, then read |

---

# BME280 Wiring

The Weather Station uses the sensor in I²C mode.

| BME280 Pin | STM32 |
|------------|-------|
| VCC | 3.3 V |
| GND | GND |
| SDA | PB7 (I2C1 SDA) |
| SCL | PB6 (I2C1 SCL) |
| CSB | 3.3 V (selects I²C mode) |
| SDO | GND or 3.3 V (selects address 0x76 / 0x77) |

---

# Integrating the Bosch Driver

The official Bosch library is intentionally hardware-independent.

Instead of calling STM32 HAL functions directly, it expects the application to
provide a small hardware abstraction consisting of three functions:

- I²C read
- I²C write
- delay function

The driver is configured like this:

```c
struct bme280_dev dev;

dev.intf     = BME280_I2C_INTF;
dev.read     = user_i2c_read;
dev.write    = user_i2c_write;
dev.delay_us = user_delay_us;
dev.intf_ptr = &hi2c1;

bme280_init(&dev);
```

This approach allows the exact same Bosch library to run on STM32, ESP32,
Arduino, Linux, or any other platform without modification.

---

# Initialization Order

The order of initialization is important.

```
Enable GPIO

↓

Initialize I²C peripheral

↓

Configure BME280 driver callbacks

↓

Call bme280_init()

↓

Start reading measurements
```

The Bosch driver immediately communicates with the sensor during
`bme280_init()`. Therefore the I²C peripheral must already be fully configured.

---

# Sensor Output

After initialization the driver returns physical values directly.

| Quantity | Unit |
|----------|------|
| Temperature | °C |
| Pressure | Pa |
| Humidity | %RH |

No manual conversion from ADC values is required because the Bosch driver
already performs all compensation calculations internally using the sensor's
factory calibration constants.

---

# Project Usage

The reusable `env_sensor` module encapsulates both the I²C communication and the
Bosch driver.

The application layer simply calls

```c
env_sensor_read();
```

and immediately receives calibrated environmental measurements without dealing
with registers, addresses, or protocol details.

This separation keeps the application independent from the sensor hardware and
makes the module reusable in future projects.
